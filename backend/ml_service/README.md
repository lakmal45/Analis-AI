# AnalisAI ML Service

This service trains and serves the signal win-probability classifier used by the futures signal engine.

## What It Does

- Trains an `XGBoost` binary classifier on exported signal samples.
- Calibrates probabilities with a chronological validation slice before live use.
- Evaluates robustness with walk-forward validation instead of a single random-style split.
- Persists versioned model artifacts and registry metadata under `ml_service/artifacts/`.
- Serves prediction, training, registry, and activation endpoints for the Node backend.

## Input Contract

The service expects the flattened feature layout generated by:

- [backend/services/mlDatasetService.js](C:/Users/LAKMAL/Desktop/github/Analis-AI/backend/services/mlDatasetService.js)

That flattening layer is the source of truth for both:

- training dataset export
- future live inference payloads

## Setup

Create a Python virtual environment and install dependencies:

```bash
cd backend/ml_service
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Export Training Data

From the backend folder:

```bash
npm run export:training-data
```

This writes:

- `backend/ml_service/data/training-data.json`
- `backend/ml_service/data/training-data.csv`

## Start The Service

```bash
cd backend/ml_service
.venv\Scripts\activate
uvicorn app:app --host 127.0.0.1 --port 8001 --reload
```

## Train A Model

With the service running:

```bash
curl -X POST http://127.0.0.1:8001/train ^
  -H "Content-Type: application/json" ^
  -d "{\"datasetPath\":\"data/training-data.csv\"}"
```

Training now uses chronological splits:

- train slice for the base model
- later calibration slice for probability calibration
- final holdout slice for untouched evaluation
- walk-forward folds for robustness checks

Automatic activation only happens when the new model clears the configured promotion thresholds.

## Lifecycle Endpoints

- `GET /health`
- `GET /models`
- `GET /training/latest`
- `POST /train`
- `POST /models/{modelVersion}/activate`
- `POST /predict`

## Predict

```json
    "context.signalType": 1,
    "context.timeframe": 60,
    "context.leverage": 10
  },
  "metadata": {
    "symbol": "BTCUSDT",
    "timeframe": "1h"
  }
}
```

## Environment Link For Node

When you begin Phase 4 integration, set this in `backend/.env`:

```env
ML_SERVICE_URL=http://127.0.0.1:8001
ML_REQUEST_TIMEOUT_MS=5000
ML_RETRAIN_INTERVAL_HOURS=24
```

`ML_RETRAIN_INTERVAL_HOURS` is optional. If it is set to a positive number, the backend will periodically export fresh training data and request a new model training run automatically.
